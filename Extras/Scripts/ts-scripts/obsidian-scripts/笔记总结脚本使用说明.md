# 📝 笔记总结脚本使用说明

## 脚本概览

本目录包含两个用于生成笔记总结报告的TypeScript脚本：

### 1. `本周笔记总结报告.ts` - 周度总结
- **功能**: 生成本周新建笔记的总结报告
- **时间范围**: 周一00:00至周日23:59
- **适用场景**: 每周回顾、知识整理

### 2. `今日笔记总结报告.ts` - 每日总结
- **功能**: 生成今日新建笔记的总结报告
- **时间范围**: 今日00:00至23:59
- **适用场景**: 每日回顾、快速浏览

## 使用方法

### 在Obsidian中运行

1. **安装CodeScript Toolkit插件**
   - 在Obsidian社区插件中搜索并安装"CodeScript Toolkit"

2. **配置脚本路径**
   - 确保脚本文件位于Obsidian库中的正确路径

3. **运行脚本**
   - 使用URL调用方式：
     ```
     obsidian://CodeScriptToolkit?module=/Extras/Scripts/ts-scripts/obsidian-scripts/本周笔记总结报告.ts
     ```
     或
     ```
     obsidian://CodeScriptToolkit?module=/Extras/Scripts/ts-scripts/obsidian-scripts/今日笔记总结报告.ts
     ```

4. **查看结果**
   - 脚本执行后会在库根目录生成对应的Markdown报告文件
   - 文件名格式：`本周笔记总结报告-YYYY-MM-DDTHH-MM-SS.md` 或 `今日笔记总结-YYYY-MM-DD.md`

```markdown
> 生成时间: 2025/8/30 00:25:28
> 时间范围: 2025年8月25日 - 2025年8月31日
> 总计: 176 篇笔记

## 📝 新建笔记列表

1. **[[笔记标题]]** - 08-30 14:25 #标签1 #标签2
   > 笔记内容摘要...

2. **[[另一个笔记]]** - 08-30 12:10 #标签3
   > 另一个笔记的摘要...
```

```markdown
   > 今日笔记内容摘要...

2. **[[今日笔记2]]** 12:10 #标签2
   > 另一个今日笔记摘要...
```

## 功能特性

### 🔍 智能内容提取
- 自动解析YAML frontmatter
- 提取笔记标签
- 生成智能摘要（移除格式符号，保留核心内容）

### 📊 统计分析
- 笔记数量统计
- 标签使用频率分析
- 时间范围过滤

### 🔗 便捷导航
- 使用`[[双链]]`格式，便于点击跳转
- 显示创建时间，便于时间排序
- 标签快速识别

## 自定义配置

### 修改时间范围
- **本周总结**: 编辑`getCurrentWeekRange()`函数
- **今日总结**: 编辑`getTodayRange()`函数

### 调整摘要长度
- 修改`generateNoteSummary()`函数中的`maxLength`参数
- 本周总结默认200字符，今日总结默认80字符

### 自定义标签显示数量
- 本周总结显示前5个热门标签
- 今日总结显示前8个标签
- 可在`generateMarkdownReport()`函数中调整

## 自动化建议

### 定时执行
- **每日总结**: 建议在每天晚上或次日早上运行
- **每周总结**: 建议在每周日晚上或周一早上运行

### 集成到工作流
- 可以结合Obsidian的模板功能
- 可以设置快捷键快速调用
- 可以集成到其他自动化脚本中

## 故障排除

### 常见问题
1. **脚本执行失败**: 检查CodeScript Toolkit插件是否正确安装
2. **路径错误**: 确认脚本文件路径正确
3. **权限问题**: 确保Obsidian有写入文件的权限

### 日志查看
- 在Obsidian开发者控制台查看详细日志
- 检查生成的报告文件是否存在
